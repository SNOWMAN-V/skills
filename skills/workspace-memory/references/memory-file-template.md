# Daily Memory Template

Use this structure when creating or refreshing a workspace memory file.

## File Name

`YYYY-MM-DD.md`

## Template

```md
# Memory for 2026-03-08
Workspace: D:\Yae_Miko

## Quick Progress
- Completed: Created the initial workspace-memory skill folder and replaced the template SKILL.md.
- In progress: Finalizing validation and polishing the memory workflow wording.
- Next step: Run the validator and test the skill on a real work session.
- Blockers: None.

## Skill structure
Explain the important structural decisions, the directories that were created, and anything that should be preserved in future edits.

## Validation
Record the commands that were run, what they verified, and any failures or follow-up actions.
```

## Section Rules

- Keep every `##` heading short and topic-driven.
- Put durable detail under the heading, not in the heading itself.
- Update the existing file when the date matches today.
- Add new sections for new important topics instead of mixing unrelated details together.

## Briefing Rules

When reading memory before a new work session, summarize only:

- Last work date
- Main completed item
- Current focus
- Active blocker or risk
- Most likely next step

If the user asks about one heading, answer from that heading's details instead of re-summarizing the full memory file.
